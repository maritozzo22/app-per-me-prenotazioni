### Step 2: Document Update Procedures

**Create**: `UPDATE_PROCEDURES.md`

**Sections**:

#### 1. Overview
- When updates are needed
- Update types (bug fixes, features, major versions)
- Update frequency recommendations

#### 2. Pre-Update Checklist
- [ ] Create backup of current app data
- [ ] Note current app version
- [ ] Charge device to at least 50%
- [ ] Ensure stable internet connection (if downloading)
- [ ] Have 30 minutes available for update process

#### 3. Update Methods

**Method 1: Automatic Backup Before Update**
1. Open app
2. Navigate to Settings → Backup & Restore
3. Tap "Create Backup"
4. Wait for completion
5. Verify backup appears in list
6. Proceed with update

**Method 2: Manual Backup**
1. Use file manager
2. Navigate to: `/Android/data/com.example.app_per_me_prenotazioni/files/backups/`
3. Copy all backup files to PC/cloud storage
4. Verify files copied successfully

**Method 3: Installing New APK**
1. Transfer new APK to device (via USB, cloud, etc.)
2. Use file manager to locate APK
3. Tap APK to install
4. If prompted, allow installation from unknown sources
5. Wait for installation
6. Open updated app
7. Verify data is intact

**Method 4: Update via Android Studio (Development)**
1. Connect device via USB
2. Open project in Android Studio
3. Build updated APK
4. Install to device
5. Test thoroughly

#### 4. Post-Update Verification
- [ ] App launches successfully
- [ ] All reservations present
- [ ] Calendar displays correctly
- [ ] Dashboard statistics accurate
- [ ] All features working
- [ ] No new issues

#### 5. Rollback Procedure (If Needed)
**If update causes problems**:

1. **Uninstall problematic version**
   - Settings → Apps → App Prenotazioni → Uninstall

2. **Install previous version**
   - Locate previous APK
   - Install APK

3. **Restore data** (if needed)
   - Open app
   - Settings → Backup & Restore
   - Select backup from before update
   - Tap Restore
   - Confirm
   - Wait for restore
   - Verify data

#### 6. Update Best Practices
- Always backup before updating
- Test new version thoroughly before relying on it
- Keep previous APK for at least 1 week
- Document any issues with new version
- Report critical issues immediately
