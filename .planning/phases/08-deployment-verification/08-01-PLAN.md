## Implementation Plan

### Step 1: Verify Current Build Configuration
**File**: `android/app/build.gradle`

**Tasks**:
- Verify target SDK version (34)
- Verify min SDK version (21)
- Check application ID: `com.example.app_per_me_prenotazioni`
- Verify all permissions are declared

**Verification Commands**:
```bash
flutter doctor -v
flutter devices
```

### Step 2: Build Fresh APK via Android Studio

**Prerequisites**:
- Android Studio installed
- JDK 11+ configured
- Android SDK 34 installed

**Build Process**:
1. Open Android Studio
2. File → Open → Select `android/` folder in project root
3. Wait for Gradle sync to complete
4. Build → Build Bundle(s) / APK(s) → Build APK(s)
5. Wait for build completion (2-5 minutes)
6. Locate APK at: `android/app/build/outputs/apk/debug/app-debug.apk`

**Expected Output**:
- APK size: ~150-160MB (debug build)
- Build successful message in Android Studio
- APK file present in output directory

### Step 3: Deploy to DN2103 Device

**Method 1: Android Studio Direct Deployment**
1. Ensure DN2103 is connected via USB
2. Verify device appears in Android Studio device dropdown
3. Click "Run" button (green play icon)
4. Wait for installation and launch

**Method 2: Command Line Deployment** (Alternative)
```bash
# Verify device connected
flutter devices

# Install APK
flutter install

# Or install specific APK
adb install android/app/build/outputs/apk/debug/app-debug.apk
```

**Expected Results**:
- App installs successfully on DN2103
- App icon appears on home screen
- No installation errors

### Step 4: Initial Smoke Test

**On-Device Verification**:
1. Launch app from home screen
2. Verify splash screen appears
3. Verify home page loads (Calendar)
4. Test bottom navigation:
   - Tap Dashboard → should load
   - Tap Reservations → should load
   - Tap Calendar → should load
   - Tap Settings → should load (dark mode toggle)
5. Test basic interaction:
   - Tap FAB (floating action button) → should open form
   - Tap back button → should navigate back
6. Verify no crashes or ANRs (Application Not Responding)

**Expected Results**:
- App launches in < 3 seconds
- All screens load without errors
- Navigation is smooth
- No visible glitches or UI issues
- No crashes during basic navigation

### Step 5: Document Build Process

**Create**: `ANDROID_BUILD_LOG.md`

**Contents**:
- Build date and time
- Android Studio version
- Gradle version
- APK size
- Device model (DN2103)
- Android version on device
- Any build warnings or issues
- Installation steps taken

## Known Issues & Solutions

**Potential Issue**: Gradle sync fails
- **Solution**: Run `flutter pub get` in project root, retry

**Potential Issue**: Device not detected
- **Solution**: Enable USB debugging on device, reinstall ADB drivers

**Potential Issue**: APK too large
- **Solution**: Expected for debug build, acceptable for personal use

**Potential Issue**: App crashes on launch
- **Solution**: Check `adb logcat` for stack trace, investigate error
