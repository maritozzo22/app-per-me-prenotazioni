## Interaction Patterns

**Add/Edit Item Flow:**
1. Tap FAB (+) → Show `AddEditInventoryItemDialog`
2. Form fields: Name (text), Category (dropdown), Quantity (number), Expiry Date (date picker, only for Alimentari), Notes (optional text)
3. Validation: Name required, Category required, Quantity ≥ 0 (initial), Expiry date required if Alimentari
4. Save → Create item + Schedule notification (if Alimentari with expiry)

**Movement Recording Flow (from D-07, D-08):**
1. Long-press item card OR tap "Registra Movimento" button → Show quantity adjustment dialog
2. Input delta (+/- integer)
3. Save → Create movement record, update item quantity

**Movement History Flow:**
1. Tap item card → Open `InventoryMovementHistorySheet`
2. Display list: Date (dd MMM yyyy, HH:mm) | Delta (+/- colored)

**Category Filter Flow:**
1. Tap FilterChip → Update provider state
2. List filters to show only selected category
3. Filter preference saved to SharedPreferences

**Expiry Notification Flow (from D-04, D-05):**
1. Item created with expiry date (Alimentari only)
2. Schedule notification for 3 days before expiry
3. Notification shows: "{ItemName} scade tra 3 giorni"
4. Tap notification → Open InventoryPage with item highlighted

**Source:** CONTEXT.md D-04 through D-10 + existing notification patterns

## Layout Specifications

**InventoryPage structure:**
```
AppBar
  Title: "Magazzino"
  Actions: [
    IconButton (settings icon) → Navigate to SettingsPage
  ]

Body (Column)
  - FilterChips row (horizontal scrollable)
    [Tutti] [Alimentari] [Tessili] [Altro]
  - Expanded: Inventory list
    - ListView.builder
    - ItemCard (leading: expiry indicator, title: name, subtitle: category + quantity, trailing: edit/delete)
    - OR EmptyStateWidget if no items

FloatingActionButton
  - Icon: Icons.add
  - Label: "Aggiungi Articolo" (extended)
  - OnTap: Show AddEditInventoryItemDialog
```

**InventoryItemCard specification:**
- Leading: `ExpiryIndicator` (colored circle 12x12dp)
- Title: Item name (`textTheme.titleMedium`)
- Subtitle: "{Category} • Qty: {quantity}" (`texttheme.bodyMedium`)
- Trailing: Row of IconButtons (edit, delete)
- onTap: Show movement history bottom sheet
- onLongPress: Show movement recording dialog

**AddEditInventoryItemDialog specification:**
- Title: "Aggiungi Articolo" (new) or "Modifica Articolo" (edit)
- Form fields (TextFormField):
  - Name: "Nome", required, maxLength: 50
  - Category: DropdownButtonFormField, options: [Alimentari, Tessili, Altro]
  - Quantity: TextFormField number keyboard, default: 1, min: 0
  - Expiry Date: TextFormField read-only, tap to show date picker, shown only if Category == Alimentari
  - Notes: TextFormField optional, maxLength: 200
- Actions: [Cancel (TextButton), Save (FilledButton)]

**Source:** Material Design 3 dialog patterns + existing form patterns from reservation forms

## Implementation Notes

**Default sort order (Claude's discretion from CONTEXT.md):**
- Primary: Expiry date (soonest first) for Alimentari
- Secondary: Name (A-Z) for Tessili, Altro
- Tertiary: Created at (newest first) as tiebreaker

**Expired items handling:**
- Always visible in list (not auto-hidden)
- Sorted to bottom when expired
- Red expiry indicator + "Scaduto il {date}" label
- Manual deletion via swipe-to-delete or delete button

**Negative quantities (from D-10):**
- Display: "Mancano: {X}" instead of "-{X}"
- Color: Red text for negative values
- Example: "Mancano: 2 asciugamani"

**Notification channel:**
- Reuse existing 'reservation_reminders' channel (simpler for user)
- Alternative: Create separate 'inventory_expiry' channel if user wants granular control
- Recommendation: Same channel for simplicity (can be split in future if requested)

**Source:** CONTEXT.md "Claude's Discretion" section + RESEARCH.md Open Questions
