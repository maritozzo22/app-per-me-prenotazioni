# Plan 07-04: Accessibility Enhancements

**Wave:** 4
**Focus:** Enhance accessibility with semantic labels, keyboard navigation, and screen reader support
**Dependencies:** Wave 1 (07-01), Wave 2 (07-02)
**Estimated Time:** 1 day
**Complexity:** Medium

## Objective

Improve accessibility throughout the app to ensure it's usable by people with disabilities, following WCAG 2.1 guidelines and Android accessibility best practices.

## Tasks

### 1. Complete Semantic Labeling (Priority: High)

#### 1.1 Form Fields
**Current State:** Some fields have Semantics, but incomplete

**Implementation:**
- Add semantic labels to all form inputs
- Include semantic hints for expected format
- Add semantic values for current input
- Mark required fields in semantics
- Link error messages to fields

**Files:**
- `lib/features/reservations/presentation/widgets/reservation_form.dart`
- `lib/features/platforms/presentation/widgets/platform_form.dart` (if exists)

**Pattern:**
```dart
Semantics(
  label: 'Guest name',
  hint: 'Enter the guest\'s full name',
  value: guestName,
  textField: true,
  onTap: () => focusNode.requestFocus(),
  child: TextField(...),
)
```

### 2. Keyboard Navigation (Priority: High)

#### 2.1 Focus Management
**Implementation:**
- Set initial focus on page load
- Logical tab order (top to bottom, left to right)
- Focus trap in modals/dialogs
- Return focus to trigger after closing modal

**Files:**
- All form pages
- All modal/dialog pages
- Settings page

**Pattern:**
```dart
AutofocusGroup(
  child: Column(
    children: [
      TextField(
        autofocus: true,
        focusNode: _focusNodes[0],
      ),
      // ... other fields
    ],
  ),
)
```

#### 2.2 Shortcuts (Web)
**Implementation:**

- Ctrl+F: Search
- Escape: Close modal/back
- Enter: Submit form

**File:** `lib/core/shortcuts/app_shortcuts.dart` (create)
